# PT Generator

AI-powered personal training session plan generator built on the Anthropic Claude API.
Produces NASM-informed, structured workout plans from a one-line session description —
available as both a **command-line tool** and a **browser-based web app**.

---

## What It Does

PT Generator solves the session-planning burden for personal trainers. Give it a client name and a one-line focus (e.g. "lower body strength, avoid knee flexion past 90°") and it returns a complete, coach-quality training plan with warm-up, main work, core/balance integration, and cool-down. Every exercise includes sets, reps, tempo, rest periods, coaching cues, regressions, and progressions.

Beyond single-session generation it also:

- **Tracks loads session-over-session** so Claude can apply progressive overload automatically
- **Manages client profiles** — constraints, preferred equipment, and machine settings (seat, lever, pad positions) saved per client
- **Records actual loads achieved** after each session via a post-workout form, feeding the next plan
- **Detects personal records** and surfaces them on the client history page
- **Generates SMART fitness goals** from history with one click
- **Suggests the next session focus** based on what was trained recently
- **Produces monthly progress summaries** written in natural language by Claude
- **Exports finished plans** to PDF, DOCX, or Markdown

## Installation

```bash
# 1. Clone the repository
git clone <repo-url>
cd pt-generator

# 2. Install dependencies
pip install -r requirements.txt

# 3. Configure environment variables
cp .env.example .env   # or create .env manually
```

Minimum `.env` for CLI use:

```
ANTHROPIC_API_KEY=your_key_here
```

Additional variables for the web app:

```
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
SECRET_KEY=a_long_random_string          # signs session cookies
DEMO_EMAIL=you@example.com              # optional — auto-seeds demo clients on first login

```

### JSON API

Authenticate with a session cookie (from web login) or an `X-Api-Key` header:

```bash
curl -X POST http://localhost:8000/api/generate \
  -H "Content-Type: application/json" \
  -H "X-Api-Key: your_api_key" \
  -d '{
    "client": "Jane Smith",
    "focus": "Lower body strength and balance.",
    "duration": 50,
    "constraints": ["avoid knee flexion past 90 degrees"],
    "equipment": ["dumbbells 5-25", "resistance bands", "stability ball"],
    "export": "pdf"
  }'
```

`export` accepts `"none"` (default), `"markdown"`, `"docx"`, `"pdf"`, or `"both"`.

| `--focus` | full-body | One-line session goal |
| `--duration` | `50` | Session length in minutes (1–180) |

| `--equipment` | profile default | Comma-separated available equipment |

| `--export` | `none` | `markdown`, `docx`, `pdf`, or `both` |

## How It Works

```
Input (client, focus, duration, constraints, equipment)
        │
        ▼
Load client profile  ──►  resolve constraints & equipment
        │
        ▼
Load session history  ──►  extract prior loads for progressive overload
        │
        ▼
Build prompt  (system: NASM coach persona  +  user: session context + prior loads)
        │
        ▼
Call Claude with tool_use  (structured JSON output enforced via schema)
        │
        ▼
Validate output against Pydantic TrainingSessionPlan schema
        │
        ▼
Persist session to history.json
        │
        ▼
Export to PDF / DOCX / Markdown  (if requested)
        │
        ▼
Return TrainingSessionPlan
```

Claude is called via the **tools API** with `TrainingSessionPlan.model_json_schema()` as the tool input schema. This guarantees structured JSON output or raises a validation error (retried automatically up to 3 times via `tenacity`).

## Configuration

Edit `app/config.py` to change the model or generation parameters:

```python
MODEL = "claude-sonnet-4-6"    # Anthropic model ID
MAX_TOKENS = 8192               # Max output tokens per generation
TEMPERATURE_GENERATE = 0.4     # Lower = more consistent, less creative
MAX_RETRIES = 3                 # Retry attempts on transient API failures
RETRY_WAIT_SECONDS = 2         # Seconds between retries
DEFAULT_DURATION = 50           # Default session duration in minutes
```

## Data Storage

No database is required. All data is stored as plain JSON files:

| File | Contents |
|------|---------|
| `data/clients/<user-id>/<slug>/profile.json` | Client name, constraints, equipment, machine settings, trainer notes |
| `data/clients/<user-id>/<slug>/history.json` | List of past sessions with date, focus, planned loads, and actual loads achieved |
| `data/clients/<user-id>/<slug>/goals.json` | SMART fitness goals (AI-generated or manual) |

Both `data/` and `outputs/` are git-ignored. Back them up separately.

## Deployment

The repo includes a `railway.toml` for one-command deployment to [Railway](https://railway.app/):

```toml
[build]
builder = "nixpacks"

[deploy]
startCommand = "uvicorn app.web.server:app --host 0.0.0.0 --port $PORT --proxy-headers --forwarded-allow-ips='*'"
healthcheckPath = "/health"
```

Set the environment variables listed in the **Installation** section as Railway service variables before deploying.
